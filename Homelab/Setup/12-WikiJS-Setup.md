# 12. Wiki.js Setup

Wiki.js runs on the Services VM with PostgreSQL as its database. All pages are stored as markdown files in a dedicated `wiki` branch of the homelab git repo, making content accessible via Obsidian as a fallback.

## Prerequisites

- Traefik routing working for Services VM (traefik-kop running)
- Git repo with a `wiki` orphan branch created
- SSH deploy key with write access to the git repo

## Git Branch Setup

Create an orphan `wiki` branch with no history from main:

```bash
git checkout --orphan wiki
git rm -rf .
echo "# Wiki" > README.md
git add README.md
git commit -m "Initial wiki branch"
git push origin wiki
git checkout main
```

The orphan flag creates a branch with no shared history - wiki content commits are completely separate from infrastructure config commits.

## Deploy Key

Generate a key pair for Wiki.js to push commits:

```bash
ssh-keygen -t ed25519 -C "wikijs" -f wikijs_deploy_key
```

Add the public key (`wikijs_deploy_key.pub`) to your GitHub repo under Settings -> Deploy keys with write access enabled.

Keep the private key (`wikijs_deploy_key`) to paste into Wiki.js storage settings.

## Compose File

Location: `docker/wikijs/docker-compose.yml` (on Services VM)

## Git Storage Configuration

After deploying, go to Administration -> Storage -> Git in the Wiki.js UI:

```
Repository URL:   git@github.com:YOURUSERNAME/YOURREPO.git
Branch:           wiki
SSH Private Key:  (paste contents of wikijs_deploy_key)
Sync direction:   Bidirectional
Sync schedule:    Every 5 minutes
```

Click Apply Changes. Wiki.js will attempt an initial sync

Verify by checking the `wiki` branch in GitHub after creating a test page.

## Authentik OIDC Setup

### In Authentik

Create an OAuth2/OIDC Provider:

- Name: Wiki.js
- Client type: Confidential
- Redirect URI: `https://wiki.lan.nathanryder.xyz/login/UUID/callback`

The UUID in the redirect URI is generated by Wiki.js per authentication strategy. To find it:

1. Go to Administration -> Authentication in Wiki.js
2. Add OpenID Connect strategy
3. Save and note the callback URL shown - it contains the UUID
4. Go back to Authentik and update the redirect URI with the full UUID callback URL

Create the Application and add to Outpost as usual.
